Section 3

Methodology

The study is implemented as a modular Python pipeline. Each stage — data, signals, backtest, metrics, visualisation, sensitivity — lives in its own module so the logic can be audited independently. This section documents every design decision that affects the reported numbers.

3.1Data & universe

The universe is five large-cap FTSE 100 constituents chosen to span different sectors, reducing the chance that any result is driven by a single industry:

Daily prices are downloaded from Yahoo Finance for 1 January 2019 to 31 December 2024. The pipeline uses auto-adjusted close prices rather than raw close. Adjusted close folds in dividends and stock splits, so the return series reflects the actual economic gain a holder receives. Raw close would inject artificial price jumps on ex-dividend dates that would trigger false crossover signals. Missing values (holidays, exchange closures, provider gaps) are forward-filled — carrying the last known price forward, the standard convention for equity series since a position's value is unchanged on a non-trading day — with a back-fill pass to handle any leading gaps.

3.2Train / test split

To prevent the conclusion from being contaminated by hindsight, the data is partitioned on 1 July 2022:

  • Warm-up / in-sample — Jan 2019 → Jun 2022 (~70% of the data). Used only to let the moving averages converge before evaluation begins.
  • Out-of-sample test — Jul 2022 → Dec 2024 (~30%). All reported performance is measured here.

Crucially, the EMAs are computed over the full price history and only the test-period slice of the resulting signal is traded. This avoids a cold-start artefact: if the EMA were started from scratch on the first test day, its early values would be unreliable and would emit spurious signals at exactly the moment evaluation begins.

3.3The exponential moving average

For a price series P and a window span n, the EMA is computed with the standard recursive formula:

α = 2 / (n + 1)
EMAt = α · Pt + (1 − α) · EMAt−1

This is computed with pandas' .ewm(span=n, adjust=False). The adjust=False choice is deliberate and matters: with adjust=True, pandas computes a batch weighted average over the entire history, which cannot be reproduced in a live system where only the previous EMA value is known. The recursive adjust=False form is exactly how EMAs are computed on Bloomberg, Reuters and most professional trading platforms. Two EMAs are built per stock — a fast 20-day and a slow 50-day line.

3.4Signal generation & look-ahead control

The raw position is long when the fast EMA sits above the slow EMA, and flat otherwise:

rawt = +1 if EMA20,t > EMA50,t, else −1
signalt = rawt−1   (shifted one day forward)
Look-ahead bias prevention A crossover is only observable at the close of day t. The signal series is therefore shifted forward one day (.shift(1)) so that a crossover seen at the close of day t only triggers a trade on day t+1. Without this shift the backtest would implicitly let the trader act on information from the future, inflating returns.

3.5The backtest engine

The portfolio is a simple two-state machine starting from £10,000:

  • Long (signal = +1): 100% of capital is invested in the stock.
  • Flat (signal = −1): 100% in cash.

Every change of state — each entry and each exit — incurs a transaction cost of 0.1% of the traded value, reflecting broker commission and bid–ask spread. On a buy, the cost is deducted from capital before shares are purchased; on a sell, it is deducted from the proceeds. The daily portfolio value is the share count times the day's price when invested, or the cash balance when flat.

Benchmark: buy-and-hold

The benchmark invests the full £10,000 at the first test-period close and never trades again. Following standard academic convention, no transaction cost is applied to this single initial purchase — its cost is negligible over a multi-year horizon. This is a passive baseline representing what an index-style investor would have earned with no active management.

3.6Performance metrics

Three metrics are computed for both the strategy and the benchmark over the test period.

Annualised Sharpe ratio

The primary risk-adjusted measure — excess return per unit of volatility, scaled to an annual figure assuming 252 trading days:

rt = Vt / Vt−1 − 1
Sharpe = ( mean(rt) − rf/252 ) / std(rt) × √252

The risk-free rate rf is set to 2.5% annualised, a blended average of the 10-year UK Gilt yield across 2019–2024.

Maximum drawdown

The largest peak-to-trough decline, the primary downside-risk measure for practitioners:

peakt = max(V0 … Vt)
MaxDD = mint ( (Vt − peakt) / peakt )

Total return

TotalReturn = (Vfinal − Vinitial) / Vinitial

3.7Reproducibility

The entire pipeline is deterministic given the price data. Parameters live in a single configuration module, and the test set is never used for tuning the headline 20/50 run. The sensitivity analysis in Section 5 sweeps a grid of fast windows {10, 20, 50} against slow windows {50, 100, 200}, skipping any pair where the fast window is not strictly shorter than the slow one.
