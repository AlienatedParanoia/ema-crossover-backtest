A.3EMA & signal generation

Two design choices matter here: adjust=False gives the standard recursive EMA used on professional platforms, and .shift(1) defers every trade to the next day so the backtest can never act on a same-day crossover (no look-ahead bias).

signals.py 
def compute_emas(price, short_window, long_window):
    short_ema = price.ewm(span=short_window, adjust=False).mean()
    long_ema  = price.ewm(span=long_window,  adjust=False).mean()
    return short_ema, long_ema

def generate_signals(price, short_window, long_window):
    short_ema, long_ema = compute_emas(price, short_window, long_window)
    raw    = (short_ema > long_ema).astype(int) * 2 - 1   # +1 long, -1 flat
    signal = raw.shift(1)                                  # trade next day -> no look-ahead
    signal.iloc[0] = 0
    return signal

A.4Backtest engine

A two-state machine: fully invested on a long signal, fully in cash otherwise. A 0.1% cost is charged on every entry and exit. Daily portfolio value is marked to market.

backtest.py — run_ema_strategy 
def run_ema_strategy(price, signals, starting_capital, transaction_cost):
    signals = signals.reindex(price.index).fillna(0)
    values  = pd.Series(index=price.index, dtype=float)
    capital, position, prev = starting_capital, 0, 0

    for date, px in price.items():
        sig = signals.loc[date]
        if sig != prev:
            if prev == 1 and position > 0:        # exit -> move to cash
                proceeds = position * px
                capital  = proceeds - proceeds * transaction_cost
                position = 0
            if sig == 1:                            # enter -> buy with all cash
                capital -= capital * transaction_cost
                position = capital / px
                capital  = 0
            prev = sig
        values.loc[date] = position * px if position > 0 else capital
    return values

A.5Performance metrics

metrics.py 
def sharpe_ratio(values, risk_free_rate):
    rets   = values.pct_change().dropna()
    excess = rets - risk_free_rate / 252
    std    = rets.std()
    return 0.0 if std == 0 else (excess.mean() / std) * np.sqrt(252)

def max_drawdown(values):
    peak = values.cummax()
    return float(((values - peak) / peak).min())

A.6How the study was run

main.py orchestrates the whole pipeline: fetch → clean → train/test split → generate signals on the full history (for EMA warm-up) → slice to the test period → run the EMA strategy and buy-and-hold → compute metrics → render charts → sweep the sensitivity grid.

Data acquisition & the rate-limit workaround

Pulling six years of daily data for five tickers tripped Yahoo Finance's rate limiter (Too Many Requests), and ordinary retries only kept the block alive. The fix was to route yfinance through a curl_cffi session that impersonates Chrome — Yahoo's throttling is largely TLS-fingerprint based, so a browser-like handshake sails through. The result is cached to data_cache/prices.csv so subsequent runs are fully offline.

Data fetch (rate-limit workaround)
